MediaRecorder


- 获取媒体流：`navigator.mediaDevices.getUserMedia`（摄像头）
    
    或 `getDisplayMedia`（屏幕）
- 创建 `MediaRecorder` 实例，传入流
- 监听 `dataavailable` 事件，收集录制的分片数据
- 调用 `start()` 开始录制 / `stop()` 结束
- 把收集到的 `Blob` 拼成文件，可：
    - 预览：`URL.createObjectURL(blob)`
    - 上传：`FormData` 传给后端
    - 下载：`<a download>`


如何录制video里的视频流 


video.captureStream()


长时间录制就是

mediaRecorder.start(10 * 1000);


然后每隔一段时间就出发这个函数 回调参数为录制的视频blob
mediaRecorder.ondataavailable


Blob 是浏览器提供的**只读二进制数据对象**，本质是一个**存储文件类型、大小和数据引用的轻量句柄**，真实二进制内容由浏览器底层管理，不直接占用 JavaScript 内存，是前端处理文件、视频、上传、下载的核心数据类型。



URL.createObjectURL

- **生成的 URL 只在当前页面生效**
    
    刷新页面就失效，不能发给后端、不能存本地。
    
- **用完一定要释放，避免内存泄漏**
    
    js
    
    ```
    URL.revokeObjectURL(url);
    ```
    
- **只能接收：Blob / File / MediaSource**
    
    不能传 base64、不能传普通字符串。
    
- **是同步 API，立即生成，非常快**



JSZip

```js
// 生成 zip blob 
const zipBlob = await zip.generateAsync({ type: 'blob' }); // 下载 
const a = document.createElement('a'); a.href = URL.createObjectURL(zipBlob); a.download = '录制视频合集.zip'; 
a.click();
```
